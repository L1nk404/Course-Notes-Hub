- **constructor** - [member function](https://www.stroustrup.com/glossary.html#Gmember-function) with the same [name](https://www.stroustrup.com/glossary.html#Gname) as its [class](https://www.stroustrup.com/glossary.html#Gclass), used to initialize [object](https://www.stroustrup.com/glossary.html#Gobject)s of its class. - **type safety** - the property that an [object](https://www.stroustrup.com/glossary.html#Gobject) can be accessed only according to its definition. [C++](https://www.stroustrup.com/glossary.html#GC++) approximates this ideal. A programmer can violate [type](https://www.stroustrup.com/glossary.html#Gtype) safety by explicitly using a [cast](https://www.stroustrup.com/glossary.html#Gcast), by using an uninitialized [variable](https://www.stroustrup.com/glossary.html#Gvariable), by using a [pointer](https://www.stroustrup.com/glossary.html#Gpointer) that doesn't point to an object, by accessing beyond the end of an [array](https://www.stroustrup.com/glossary.html#Garray), and by misusing a [union](https://www.stroustrup.com/glossary.html#Gunion). For low-level systems code, it can be necessary to violate type safety (e.g. to write out the [byte](https://www.stroustrup.com/glossary.html#Gbyte) representation of some objects), but generally type safety must be preserved for a program to be correct and maintainable.
